SikshaSetu Production LMS: Algorithm Documentation

1. Weighted Scoring Algorithm (Mastery Calculation)

Purpose:

To provide a high-integrity measure of student progress by balancing unique content consumption (participation) with cognitive validation (performance).

Formula:

M = (P * 0.6) + (Q * 0.4)

Variables:

  • M (Mastery Score): Normalized competence score (0–100%).
  • P (Progress Score): Unique Watch Coverage percentage (Unique Seconds / Total Seconds).
  • Q (Quiz Score): Assessment performance percentage (Correct Attempts / Total Questions).

Process:

  1. Tracking of [start, end] video segments at the frontend.
  2. Interval merging at the backend to compute true unique coverage (monotonic).
  3. Verification of MCQ attempts across all course lessons.
  4. Weighted sum calculation with a final "Never Decreases" guard.

Example:

A student watches 80% of a 1-hour course uniquely and scores 90% on quizzes.
M = (80 * 0.6) + (90 * 0.4) = 48 + 36 = 84% (Mastery Achieved)

Edge Cases:

Admin/Staff accounts are excluded from progress updates. Courses without quizzes default to a 100% quiz score once progress is complete.

2. Hacker News Gravity Algorithm (Trending)

Purpose:

To rank courses based on platform-wide momentum rather than lifetime counts, ensuring a fresh discovery feed for students.

Formula:

Score = [(Enrollments*3) + (Views*1) + (Likes*2)] / (Age_Hours + 2)^1.5

Variables:

  • Enrollments (Weight: 3): High-intent conversion metric.
  • Views (Weight: 1): High-volume discovery metric.
  • Likes (Weight: 2): Social validation metric.
  • Age_Hours (T): Hours since publication.
  • 1.5 (G): Gravity/decay constant.

Process:

  1. Extraction of real-time interaction metrics (Real Data Layer).
  2. Age calculation in hours from published_at metadata.
  3. Exponential decay application toengagement sum.
  4. Global ranking and sorting.

Why This Approach:

It solves the "Stale Dashboard" problem by allowing new high-quality courses to outrank older courses that have stopped growing.

3. Content-Based Filtering Algorithm (Recommendations)

Purpose:

To deliver a personalized learning experience by suggesting content that overlaps with the user's established category and interest profile.

Formula:

S = sum(w(Tag_Match)) + Category_Match_Signal

Variables:

  • User Profile: Set of Tag IDs and Category IDs from enrolled courses.
  • Candidate Set: Published courses not yet enrolled or authored by the user.
  • Tag Match: Primary scoring signal for semantic similarity.

Process:

  1. Database aggregation of user interests from enrollment history.
  2. Constraint filtering (exclude owned/authored courses).
  3. Similarity scoring via tag overlap annotation.
  4. Chronological fallback for secondary tie-breaking.

Why This Approach:

Handles the "Cold Start" problem for items — a new course can be recommended immediately based on its tags, even without initial user-item interaction data.
